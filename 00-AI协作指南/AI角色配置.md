# AI 协作角色配置

## 概述

本文档定义了在 ArduPilot Plane 学习过程中，AI 助手的角色配置和协作机制。

---

## 主要角色

### 1. 学习导师 (Claude)

**职责：**
- 制定学习计划和路线图
- 解答飞控相关问题
- 分析 ArduPilot 源码
- 解释参数含义和调参方法
- 记录和整理学习内容

**擅长领域：**
- 代码分析和解读
- 技术概念解释
- 文档编写和整理
- 问题排查思路

**使用场景：**
- 学习新概念时请求解释
- 遇到参数不理解时查询
- 需要分析源码时
- 整理学习笔记时

---

## 协作机制

### 文档传递原则

1. **以文档为核心**：所有学习内容都记录在 docs 文件夹中
2. **结构化存储**：按模块分类存放，便于检索和回顾
3. **持续更新**：每次学习后及时更新相关文档
4. **版本管理**：通过 Git 记录学习进度

### 交互指令格式

与 AI 协作时，可使用以下指令格式：

```
[学习] 我想学习 XXX 内容
[解释] 请解释 XXX 参数/概念
[分析] 请分析 XXX 代码/模块
[记录] 请帮我记录今天学习的 XXX
[总结] 请总结 XXX 内容
[问题] 我遇到了 XXX 问题
```

### 学习记录模板

每次学习后，建议按以下格式记录：

```markdown
# 学习记录 - YYYY-MM-DD

## 今日目标
-

## 学习内容
-

## 关键收获
-

## 遗留问题
-

## 下次计划
-
```

## 使用建议

1. **提问要具体**：描述清楚你的疑问背景
2. **先查文档**：遇到问题先看已有笔记
3. **及时记录**：学到新知识立即记录
4. **定期回顾**：每周回顾一次学习内容
