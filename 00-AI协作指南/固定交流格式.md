# 固定交流格式

## 使用说明

从下周一开始，所有与 C老师的交流必须使用本模板。这样能保持上下文一致、降低遗漏、提高总结质量。

确认流程（所有场景通用）：
- C老师回复后必须给出【请确认】
- 你需要回复【确认】或【更正：...】后再继续下一步

---

## 每日开始（3 行启动）

```
C老师，【今日目标】
1) …
2) …
【预计时长】X 小时
【输出物】要更新的文档/截图/代码
【表达优化】请你优化我的目标表达后再继续
```

---

## 学习中提问（快速定位）

```
C老师，【问题】
【背景】我在做…
【现象】出现…
【已尝试】…
【期望】我想要…
【表达优化】请你优化我的提问表达后再回答
```

C老师输出：

```
【解答】
【可执行步骤】
【请确认】请你确认或更正后再继续
```

---

## 每日结束（固定 5 格）

```

【关键结论】
【下一步】
【表达优化】请你优化我的总结表达后再继续
```

C老师输出：

```

【纠错/补充】
【知识卡片】
【建议行动】
【提交状态】仅同步 docs，确认无敏感信息
【请确认】请你确认或更正后再继续
```

## 每日结束后的同步要求（仅 docs）

学习结束后，按以下步骤同步到远程仓库（只提交 `docs`）：

```
cd /e/A_MY_Projects/飞控参数学习包/docs
git status
git add .
git commit -m "chore: daily notes"
git push
```

注意事项：
- 只在 `docs` 目录执行提交，避免提交其它文件
- 禁止提交账号、密码、Token、证书等敏感信息
